# Prompt: Arquitectura Angular + Estado + Design System

Actúa como un **arquitecto de software senior especializado en Angular** y diseña la base de un proyecto frontend moderno, escalable y mantenible.

---

## 1. Arquitectura del proyecto
Define una arquitectura clara y profesional que incluya:
- Estructura de carpetas recomendada (feature-based o domain-driven).
- Separación de responsabilidades:
  - **Core**: servicios globales, interceptores, guards.
  - **Shared**: componentes reutilizables, pipes, directivas.
  - **Features**: módulos organizados por dominio.
- Implementación de **lazy loading**.
- Convenciones de nombres (archivos, componentes, servicios).
- Estrategia de deteccion de cambios ( changeDetection:ChangeDetectionStrategy.OnPush)
- Buenas prácticas de organización del código.
---
## 2. Manejo de estado
Recomienda y justifica la mejor estrategia moderna:
- Opciones:
  - SignalStore
  - Signals (Angular)

Incluye:

- Cuándo usar cada enfoque.
- Ventajas y desventajas.
- Ejemplo básico de implementación.
- Manejo de efectos (side effects).
- Organización del store:
  - actions
  - reducers
  - selectors
- Estrategia de escalabilidad.
